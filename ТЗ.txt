Архитектура — Docker-сервисы
:8000
web — FastAPI приложение
Основное приложение: рендер страниц (SSR Jinja2), REST API для фронта, бизнес-логика. Точка входа: uvicorn main:app

:5432
db — PostgreSQL 16
База данных. Volume для персистентности данных. Доступна только внутри Docker-сети.

:80 / :443
nginx — Reverse proxy
Проксирует запросы на FastAPI. Отдаёт статику (/static/, /media/) напрямую. SSL через Let's Encrypt (Certbot).

worker
worker — Background tasks
Отправка уведомлений (email, Telegram, MAX). Запускается отдельным процессом. Можно через FastAPI BackgroundTasks или отдельный контейнер с очередью задач (asyncio queue).

# docker-compose.yml (упрощённая структура)
services:
  db:
    image: postgres:16-alpine
    volumes: [pgdata:/var/lib/postgresql/data]
    environment: { POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD }

  web:
    build: .
    command: uvicorn main:app --host 0.0.0.0 --port 8000
    depends_on: [db]
    env_file: .env
    volumes: [./media:/app/media]

  nginx:
    image: nginx:alpine
    ports: ["80:80", "443:443"]
    depends_on: [web]
    volumes: [./nginx.conf, ./certbot]

volumes: { pgdata }

Расписание доставки — слоты и дедлайны
Часовой пояс
Все операции в UTC+7 (Томск). Хранить и сравнивать только в UTC+7.
Тип 1 — Дефолтный
Привязан к дню недели (1–7). Повторяется каждую неделю. Несколько записей на один день — разрешено.
Тип 2 — Конкретная дата
Приоритет выше дефолтного. Перекрывает дефолт для указанной даты полностью.
Слоты
Диапазон доставки / интервал = список слотов. Пример: 19:00–21:00 / 1ч = [19–20, 20–21]
Дедлайн
deadline_dt = delivery_date − deadline_days_before (days) + deadline_time
Варианты дедлайна
В тот же день (0), за день (1), за 2 дня (2), за 3 дня (3)
День без записей
= выходной. Не отображается в пикере дат.
Алгоритм: для каждой из ближайших 14 дат: 1) ищем записи по конкретной дате → если есть, используем только их; 2) иначе — дефолт дня недели; 3) для каждой записи проверяем now(UTC+7) ≤ deadline_dt; 4) генерируем слоты из диапазона. Возвращаем даты с флагом has_slots.

Метод
Только телефон + OTP. Без паролей. Первый вход = авторегистрация.
Каналы OTP
Пользователь выбирает: MAX (через MAX Business API) или Telegram-бот

JWT в httpOnly cookie, TTL 30 дней.

Deeplink → /start в боте → бот получает chat_id → привязывает к аккаунту по токену.

Зоны доставки
Хранение
GeoJSON-полигоны в PostgreSQL (поле JSON). Каждая зона — замкнутый полигон.
Определение зоны
Python-библиотека Shapely:
Point(lon, lat).within(Polygon(coords))
Несколько зон
Если точка в нескольких — берётся зона с наибольшим priority.
Вне зон
Сообщение «Доставка в ваш район пока недоступна».
Параметры зоны
Название, цена доставки, мин. сумма для бесплатной доставки, мин. сумма заказа.
13
Оплата — ЮКасса + наличные + терминал
ЮКасса
SDK yookassa. При создании заказа → создаём Payment → редирект на confirmation_url.
Webhook
POST /api/payment/callback → проверяем подпись → обновляем payment_status = 'paid' → начисляем кешбэк → уведомляем клиента.
Терминал / наличные
Заказ создаётся сразу. payment_status = 'pending'. Оператор подтверждает оплату вручную в админке.
Отмена онлайн
Клиент отменяет → API ЮКассы refund → payment_status = 'refunded'.
14
Промокоды и кешбэк
Тип 1 — Скидка %
Скидывает X% с суммы заказа. Настройки: %, период, лимит использований, мин. сумма заказа.

Тип 2 — Кешбэк покупателю
После доставки начисляется X% от суммы заказа на кешбэк-счёт покупателя.

Тип 3 — Реферальный
Покупатель получает X% кешбэка. Владелец кода (блогер) получает Y% с каждой покупки по его коду.

Трата кешбэка
До
cashback_max_pct
% от суммы заказа (глобальная настройка в site_settings).
Начисление
Только после статуса «Доставлен». Не при создании заказа.
Вывод (блогер)
Оператор вручную уменьшает баланс в админке → тип транзакции «withdraw» с комментарием.
История
Полный лог cashback_transactions — виден пользователю в ЛК и оператору в админке.

Жизненный цикл заказа
Статус	Кто меняет	Уведомление клиенту
🆕 Новый	Автоматически при оформлении (ЮКасса — после webhook)	EmailTGMAX «Заказ №X принят! Доставим [дата] [слот]»
✅ Принят	Оператор вручную в админке	EmailTGMAX «Заказ №X принят в работу. Готовим!»
🚗 Доставляется	Оператор — когда курьер выехал	EmailTGMAX «Курьер уже едет! Прибудет [слот]»
✓ Доставлен	Оператор. Триггер кешбэка.	EmailTGMAX «Доставлен! Приятного аппетита 🍽 +N₽ кешбэка»
✕ Отменён	Клиент (только «Новый») или оператор (любой статус)	EmailTGMAX «Заказ №X отменён. [причина]. Возврат 3–5 дней»
При отмене: использованный кешбэк возвращается на баланс. Счётчик использований промокода decrements. Онлайн-оплата → автовозврат через ЮКассу API.
